# REFACTOR_PROMPT 031: Generate Assigner Recipes via Python Script

## Objective

Automate the creation of `.ws1-assigner.recipe.yaml` files for all existing `.ws1.recipe.yaml` recipes in the `ws1-recipes/` directory. This enables teams to manage app assignments independently from the import/upload workflow, supporting the [cloud-autopkg-runner (CAR)](https://pypi.org/project/cloud-autopkg-runner/) refactoring initiative.

## Background: Role of Assigner Recipes

### Why Separate Assigner Recipes?

Following the successful introduction of `WorkSpaceOneAssigner.py` processor (see REFACTOR_PROMPT 030), assigner recipes allow:

1. **Workflow Decoupling**: Separate app upload (import) from smart group assignment (assigner)
2. **Independent Scheduling**: Update assignments without re-uploading app packages (faster, less network overhead)
3. **Flexible Integration**: CAR environments can chain import + assignment separately or run assignments standalone
4. **Operational Flexibility**: Administrators can adjust assignments post-deployment without reprocessing downloads/packages
5. **Idempotent Assignments**: Leverage `#AUTOPKG` / `#AUTOPKG_DONE` tagging to safely re-run assignments (no duplicates)

### Example Use Cases

- **Initial Deployment**: Import app once, then run separate assignment recipe multiple times with different smart groups without re-uploading
- **Scheduled Updates**: Run pruner recipe weekly, assigner recipe monthly to maintain smart group membership
- **Multi-Stage Rollout**: Import to testing group, wait for validation, then use assigner recipe to promote to production group
- **Post-Import Fixes**: If assignment logic changes (e.g., add new department group), regenerate assignments without full package import

### Key Differences: `.ws1.recipe.yaml` vs. `.ws1-assigner.recipe.yaml`

| Aspect | `.ws1.recipe.yaml` | `.ws1-assigner.recipe.yaml` |
|--------|--------|--------|
| **Purpose** | Download app, build package, upload to WS1, assign to groups | Update smart group assignments only |
| **Processor** | `com.github.codeskipper.OMNISSA-WorkSpaceOneImporter/WorkSpaceOneImporter` | `com.github.codeskipper.OMNISSA-WorkSpaceOneAssigner/WorkSpaceOneAssigner` |
| **ParentRecipe** | **Yes** — chains community munki recipe (e.g., `com.github.autopkg.munki.firefox-rc-en_US`) | **No** — standalone, no download/packaging |
| **App-specific inputs** | NAME, LOCALE, RELEASE, DISPLAY_NAME, MUNKI_REPO_SUBDIR, pkginfo | NAME only |
| **ws1_app_assignments** | Optional (performed after upload) | Optional (key focus of recipe) |
| **Typical runtime** | ~1–5 minutes (large network I/O) | <30 seconds (API calls only) |
| **Typical schedule** | On app version change (triggered by download) | Weekly/monthly (independent interval) |

### Recipe Metadata Fields

For **each** assigner recipe to generate, you need the following mapping:

```python
(
    filename,                # Name of .ws1-assigner.recipe.yaml file (e.g., "Firefox.ws1-assigner.recipe.yaml")
    app_display_name,        # Human-readable app name in NAME input field (e.g., "Mozilla Firefox")
    identifier_suffix,       # Suffix for Identifier (CamelCase, no spaces; e.g., "MozillaFirefox")
    description_name         # Description field name (e.g., "Mozilla Firefox")
)
```

#### 4. **Example Script Structure (Pseudocode)**

```python
#!/usr/bin/env python3
"""Generate assigner recipes for all ws1 recipes. Kept in repository for reference and future reuse."""

import os

# Script location: ws1-processors/dev work/2026 refactoring to separate WS1 operations/
# Output recipes to: ws1-recipes/ (at repo root level)
BASE = os.path.join(os.path.dirname(os.path.abspath(__file__)), "../../../ws1-recipes")

# YAML Template (multiline string with .format() placeholders)
TEMPLATE = (
    "Comment: |\n"
    "  recipe with ws1_assignments for AppsV2 API style assignment-rules\n"
    "  See https://as135.awmdm.com/API/help/#!/apis/10001?!%2FAppsV2%2FAppsV2_UpdateAssignmentRuleAsync\n"
    "Description: Assigns {description_name} to smart groups in WorkSpace ONE UEM.\n"
    # ... (rest of template from reference recipe)
)

# Recipe metadata: (filename, app_display_name, identifier_suffix, description_name)
recipes = [
    ("Apparency.ws1-assigner.recipe.yaml", "Apparency", "Apparency", "Apparency"),
    ("CitrixWorkspace.ws1-assigner.recipe.yaml", "Citrix Workspace", "CitrixWorkspace", "Citrix Workspace"),
    # ... (all 25 recipes)
]

# Generate each recipe file
for filename, app_display_name, identifier_suffix, description_name in recipes:
    path = os.path.join(BASE, filename)
    content = TEMPLATE.format(
        app_display_name=app_display_name,
        identifier_suffix=identifier_suffix,
        description_name=description_name
    )
    with open(path, "w") as f:
        f.write(content)
    print(f"Created: {filename}")

print(f"\nTotal: {len(recipes)} assigner recipes created.")
```
