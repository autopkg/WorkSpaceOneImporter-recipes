## Objective

Move the `_resolve_ogid()` and `_search_apps()` methods from `WorkSpaceOnePruner.py` (where they were private methods) into `WorkSpaceOneImporterBase.py` as public instance methods, since they are needed by multiple processors (`WorkSpaceOnePruner` and `WorkSpaceOneImporter`).

---

## Rationale

Both `WorkSpaceOneImporter` and `WorkSpaceOnePruner` need to:
1. Resolve a textual Organization Group ID to a numeric ID via the WS1 API
2. Search for existing app versions on the WS1 server

Previously these were:
- Inline code in `WorkSpaceOneImporter.ws1_import()` (lines 274â€“307)
- Private methods `_resolve_ogid()` and `_search_apps()` in `WorkSpaceOnePruner`

By promoting them to the base class, all current and future processors inherit these capabilities without code duplication.

## Files Modified

### 1. `ws1-processors/ws1_lib/WorkSpaceOneImporterBase.py`

**Added two public methods to the `WorkSpaceOneImporterBase` class:**

#### `resolve_ogid(self, api_base_url, org_group_id, headers_v2)`
- Resolves a textual WS1 Organization Group ID to a numeric ID
- Calls `GET /api/system/groups/search?groupid={org_group_id}`
- Raises `ProcessorError` on failure
- Error messages are processor-neutral (no `"WorkSpaceOnePruner:"` or `"WorkSpaceOneImporter:"` prefix)

#### `search_apps(self, api_base_url, ogid, app_name, headers)`
- Searches WS1 for existing versions of a given app
- Calls `GET /api/mam/apps/search?locationgroupid={ogid}&applicationname={name}`
- Returns the JSON response dict, or `None` if no apps found (status != 200)
- Raises `ProcessorError` if the request throws an exception

Both methods include proper docstrings with Args, Returns, and Raises sections.

## Design Decisions

| Decision | Rationale |
|---|---|
| **Public methods** (not private `_` prefixed) | These are intended to be called by subclass `main()` methods â€” they are part of the base class's supported API. |
| **Processor-neutral error messages** | Error messages don't include a processor name prefix since they live on the base class. The calling processor's context will be evident from the traceback. |
| **`search_apps()` returns `None` for non-200** | Clean API: callers check for `None` instead of inspecting HTTP status codes. Consistent with the pattern already used in `WorkSpaceOnePruner`. |
| **`else:` replaces `elif r.status_code == 204:`** | The old code only handled 200 and 204. Any non-200 status (including 204) from `search_apps()` returns `None`, so the `else` branch covers all "app not found" cases. |
