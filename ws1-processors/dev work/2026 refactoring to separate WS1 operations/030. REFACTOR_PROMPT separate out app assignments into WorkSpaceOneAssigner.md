# REFACTOR_PROMPT 030: Separate App Assignments (V1 & V2 APIs) into WorkSpaceOneAssigner Processor

## Objective

Extract the app assignment functionality from `WorkSpaceOneImporter.py` into a new dedicated AutoPkg processor called `WorkSpaceOneAssigner.py`. This processor will manage both WS1 REST API V1 and V2 app assignments independently, supporting the [cloud-autopkg-runner (CAR)](https://pypi.org/project/cloud-autopkg-runner/) refactoring initiative.

## Current State

### Existing Methods in WorkSpaceOneImporter (to be extracted)

1. **`ws1_app_assignments()`** (lines ~400–615)
   - Handles API V2 assignment rules (PUT `/api/mam/apps/{applicationUuid}/assignment-rules`)
   - Manages assignment priority, effective dates, deployment delays
   - Handles `#AUTOPKG` and `#AUTOPKG_DONE` tagging in assignment descriptions
   - Skips assignments with future effective dates
   - Reports new assignment rules in summary result

2. **`ws1_app_assignment_conf()`** (lines ~617–632)
   - Assembles app assignment dict for API V1 calls
   - Maps push mode ("Auto" / "On-Demand") to `MacOsDesiredStateManagement` boolean
   - Constructs `DeploymentParameters` struct

3. **`ws1_app_assign()`** (lines ~634–657)
   - Calls WS1 API V1 POST `/api/mam/apps/internal/{applicationId}/assignments`
   - Handles errors and sets `ws1_app_assignments_changed` env var

4. **`get_smartgroup_id()`** (lines ~168–190)
   - Searches WS1 for smart group by name
   - Returns `(sg_id, sg_uuid)` tuple
   - Handles URL encoding of spaces in group names

### Current Integration Points

- Called from `ws1_import()` method at lines ~373–377 (V1) and ~385–387 (V2)
- Uses environment variables:
  - `ws1_app_assignments` (advanced V2 assignments)
  - `ws1_smart_group_name` (simple V1 assignment)
  - `ws1_push_mode` (simple V1 assignment mode)
  - `ws1_update_assignments` (boolean flag to enable assignment updates)
  - `ws1_console_url` (for generating console links in summary)
  - `NAME`, `ws1_app_id`, `ws1_imported_new` (inherited from processor env)
- Sets output variables:
  - `ws1_app_assignments_changed` (True if assignments were modified)
  - Updates `ws1_importer_summary_result` dict with assignment details

## Design: WorkSpaceOneAssigner Processor

### Overview

New file: `ws1-processors/WorkSpaceOneAssigner.py`

- **Inherits from:** `WorkSpaceOneImporterBase` (for auth & TLS)
- **Purpose:** Standalone app assignment management for both API V1 and V2
- **Targets:** Can be used in recipes to update assignments for existing apps without re-uploading
- **CAR Compatibility:** Enables cloud-autopkg-runner workflows to manage assignments separately from uploads

### Input Variables

```python
input_variables = {
    **WorkSpaceOneImporterBase.input_variables,
    "ws1_app_id": {
        "required": True,
        "description": "Application ID of the app in WS1 UEM to assign.",
    },
    "ws1_app_uuid": {
        "required": False,
        "description": "Application UUID of the app in WS1 UEM (for API V2 calls). Auto-resolved from app_id if not provided.",
    },
    "ws1_console_url": {
        "required": False,
        "description": "Base url of WorkSpace ONE UEM Console server for result lookup (e.g., https://admin-mobile.myorg.com)",
    },
    "ws1_smart_group_name": {
        "required": False,
        "description": "The name of the first smart group the app should be assigned to (simple V1 assignment).",
    },
    "ws1_push_mode": {
        "required": False,
        "description": "For a simple app assignment, how to deploy the app: 'Auto' or 'On-Demand' (V1 only).",
    },
    "ws1_app_assignments": {
        "required": False,
        "description": 'Advanced app assignments (API V2 rules). Complex structure—see wiki for format.',
    },
}
```

### Output Variables

```python
output_variables = {
    "ws1_app_assignments_changed": {
        "description": "True if any app assignments were successfully applied.",
    },
    "ws1_importer_summary_result": {
        "description": "Dictionary with summary of assignment changes for reporting.",
    },
}
```

### Core Methods (extracted & adapted)

1. **`get_smartgroup_id()`** — Unchanged (helper)
2. **`ws1_app_assignment_conf()`** — Unchanged (helper)
3. **`ws1_app_assign()`** — Unchanged (API V1)
4. **`ws1_app_assignments()`** — Adapted for:
   - Fetch app UUID from `ws1_app_id` if `ws1_app_uuid` not provided
   - Simplified: no need to check `ws1_imported_new` flag (assume app exists)
   - Return assignment count for reporting
5. **`main()`** — New orchestration:
   - Initialize TLS & auth headers
   - Resolve OGID from `ws1_groupid`
   - Branch logic:
     - If `ws1_smart_group_name` specified → use V1 assignment (call `ws1_app_assign()`)
     - Else if `ws1_app_assignments` specified → use V2 assignment (call `ws1_app_assignments()`)
     - Else → log warning & return
   - Build `ws1_importer_summary_result` for reporting
   - Set `ws1_app_assignments_changed` output

### Key Differences from WorkSpaceOneImporter

| Aspect | WorkSpaceOneImporter | WorkSpaceOneAssigner |
|--------|----------------------|----------------------|
| **Scope** | Upload + assign | Assign only |
| **`ws1_app_id`** | Output (generated) | Input (required) |
| **`ws1_imported_new`** | Set by processor | Not used (assume exists) |
| **Skip future dates** | Yes (only update on day0+) | Yes (consistent logic) |
| **#AUTOPKG tag logic** | Preserve; skip if #AUTOPKG_DONE | Same (for idempotency) |
| **Integration** | End of `ws1_import()` | Standalone in `main()` |

## Code Patterns & Conventions

### Preserve Existing Patterns

- **Keychain & TLS**: Inherited from WorkSpaceOneImporterBase (no changes needed)
- **Verbose logging levels**:
  - Level 1: Important workflow steps
  - Level 2: API details & intermediate results
  - Level 3: Config dumps
  - Level 4: Sensitive data (tokens, full API responses)
- **Error handling**: Raise `ProcessorError` for recoverable issues; let requests exceptions propagate for logging
- **Environment variables**: Prefix input vars with `ws1_` (lowercase); use `self.env.get()` with optional defaults
- **DateTime handling**: Use `datetime.fromisoformat()` for ISO 8601 strings; account for Z/timezone in responses

### Input Variable Validation

- Check `ws1_app_id` is a valid int/string before use
- Validate `ws1_smart_group_name` or `ws1_app_assignments` is provided (enforce mutual exclusivity)
- Validate `ws1_push_mode` is "Auto" or "On-Demand" if V1 assignment is used

## Edge Cases & Considerations

1. **App UUID resolution**: If `ws1_app_uuid` not provided, fetch via GET `/api/mam/apps/internal/{ws1_app_id}`
2. **Existing assignments**: `ws1_app_assignments()` already handles checking #AUTOPKG tags; maintain logic
3. **V1 vs V2 conflict**: If both `ws1_smart_group_name` AND `ws1_app_assignments` are provided, log error & raise ProcessorError
4. **Console URL**: Optional for link generation; gracefully handle missing/invalid URLs
5. **idempotency**: Assignments with #AUTOPKG_DONE tag should be skipped on re-run (already in code)

- Phase 1 (create standalone processor) completed. Phase 2 (refactor WorkSpaceOneImporter to delegate) deferred to avoid breaking existing recipes.

- `ws1_app_ass_day0` initialized to `datetime.today().date()` at the top of V2 method to avoid potential reference-before-assignment warning.
