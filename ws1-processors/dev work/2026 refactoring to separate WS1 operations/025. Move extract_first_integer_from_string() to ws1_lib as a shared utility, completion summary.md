## Objective

Move the `extract_first_integer_from_string()` utility function from individual processor files into the `ws1_lib` shared library to follow the DRY principle and enable code reuse across all WS1 processors.

---

## Background

The function `extract_first_integer_from_string()` was previously defined in:
- `WorkSpaceOneImporter.py` (lines 75-81)
- `WorkSpaceOnePruner.py` (lines 42-47)

This duplicate implementation violated the DRY principle and made maintenance harder. Moving it to the shared library creates a single source of truth.

**Implementation:**
```python
def extract_first_integer_from_string(s):
    """Search for the first occurrence of a sequence of digits and return as int."""
    match = re.search(r"\d+", s)
    if match:
        return int(match.group())
    return None
```

**Location:** Module-level utility function, consistent with other helpers like `is_url()`, `get_timestamp()`, etc.

## Benefits of This Refactoring

1. **Single Source of Truth**
   - One canonical definition of the utility function
   - Easier to locate and modify

2. **Code Reusability**
   - Future processors can import without duplication
   - Consistent behavior across all processors

3. **Easier Maintenance**
   - Bug fixes only need to be made in one place
   - Reduces cognitive load when updating the function

4. **Better Code Organization**
   - Utilities are centralized in `ws1_lib`
   - Clear separation between shared utilities and processor-specific logic

5. **Public API**
   - Function is now part of the documented `ws1_lib` public API
   - Accessible via `from ws1_lib import extract_first_integer_from_string`
