# Deprecate and Remove `ws1_app_versions_prune()` from `WorkSpaceOneImporter`
## Completion Summary

**Date:** May 20, 2026
**Prompt:** 027
**Status:** ✅ COMPLETED

---

## Objective

Remove the `ws1_app_versions_prune()` method and its related input variables from `WorkSpaceOneImporter.py`, since pruning has been fully migrated to the standalone `WorkSpaceOnePruner` processor and tested successfully in production.

---

## Rationale

- The pruning logic was extracted to `WorkSpaceOnePruner.py` (Prompt 012) and has been running successfully in production as a pre-processor.
- Keeping the duplicate code in `WorkSpaceOneImporter` was only for backwards compatibility during the migration period.
- All recipes have been migrated to use `WorkSpaceOnePruner` as a separate processor step.
- The migration is complete — the duplicate code can now be removed.

---

## Migration Note for Recipe Authors

Recipes that previously relied on `WorkSpaceOneImporter` doing pruning (via `ws1_app_versions_prune: "True"` in the importer's Arguments) **must** now use `WorkSpaceOnePruner` as a separate processor step. The recommended pattern is:

```yaml
Process:
- Processor: com.github.codeskipper.VMWARE-WorkSpaceOnePruner/WorkSpaceOnePruner
  Arguments:
    ws1_app_versions_prune: "True"
    ws1_app_versions_to_keep: '%WS1_APP_VERSIONS_TO_KEEP%'
    # ... auth arguments ...

- Processor: com.github.codeskipper.VMWARE-WorkSpaceOneImporter/WorkSpaceOneImporter
  Arguments:
    # ... import arguments (no pruning args needed) ...
```

If a recipe still passes `ws1_app_versions_prune` to `WorkSpaceOneImporter`, it will simply be ignored (unrecognised input variable — AutoPkg does not error on extra arguments).
